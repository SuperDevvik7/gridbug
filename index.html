GridBug

A two-player arcade maze game built from the ground up on an FPGA, rendered on a 32×32 RGB LED matrix and housed in a handcrafted cardboard arcade cabinet.

The Game

Collect. Evade. Survive.

GridBug is a Pac-Man inspired head-to-head duel. Player 1 races through a maze trying to collect every pellet on the board. Player 2 has one job — catch Player 1 before all the pellets are gone.

If Player 1 clears the board, they win. If Player 2 catches Player 1 by overlapping their position, Player 2 wins. It's a tense game of cat and mouse on a tiny glowing grid.

The Hardware

FPGA-Powered Arcade

The entire game runs on an Alchitry Au FPGA programmed in Lucid HDL. There is no software, no CPU, no operating system — just raw digital logic executing every game tick in a single clock cycle at 100 MHz.

The display is a 32×32 HUB75 RGB LED matrix driven by a custom matrix driver. Player inputs come from arcade-style push buttons with edge detection. The whole thing sits inside a hand-built arcade cabinet complete with a “Insert Token Here” slot and Pac-Man artwork.

Under the Hood

Custom Datapath & FSM

The game logic is entirely combinational — every button press triggers a full pipeline of position computation, boundary checks, wall collision detection, pellet intersection, win condition evaluation, and state updates, all within a single clock edge.

A separate registered FSM handles pixel rendering, sweeping through all 1024 pixels to write the correct colour (player, wall, pellet, or background) into a dual-port RAM that feeds the LED matrix driver continuously.
